## 0. 一句话

现在的 bar 不是"墨底绢纱"，而是一条**被壁纸染脏的暖灰带**，信息又被甩成左右两坨、中间留一段死白。重构的目标不是换皮，是把这条带子重新**装裱成一条横幅（横披），借用立轴的钤印与款识章法**：沉墨为绫、双线为杆、左钤引首与姓名章、中段留白如未着墨的绢、右落状态款识与押角印。

## 1. 诊断：病在哪里（带证据）

设计不是从"我觉得不好看"开始的，是从像素开始的。对当前桌面截图（`import -window root`）取色，立轴的"骨"和"章法"两处病被坐实。

### 1.1 墨骨没了 —— 这是 bug，不是审美

| 采样点 | 实测 RGB | 明度 | 冷暖 (R−B) |
|---|---|---|---|
| bar 底色（x=400/700/1200/1700, y=12，整条均匀） | `(82,76,70)` | **76** | **+12 偏暖** |
| 设计意图墨底 `bg_base #1a1a1e` | `(26,26,30)` | 28 | −4 冷 |
| bar 正下方模糊壁纸（x=1200, y=400） | `(63,61,69)` | 62 | −6 冷 |

两个致命点：

1. **层级反了**。bar 明度 76，**比它该压住的壁纸（62）还亮 14 阶**。装裱的绫边应当是全屏最沉的层，现在它比画心还浮。
2. **冷暖反了**。bar 偏暖（R−B = +12），墨底本是冷灰（R−B = −4）。暖色是透上来的。

反推那 25% 透色（由 `result = α·ink + (1−α)·wallpaper` 解出）≈ `(250,226,190)`，正是秋叶壁纸顶部的浅暖高光。

**根因**：§2.2 的 L1 = 0.75 是在**深色壁纸假设**下调的，对浅色 / 高光壁纸不鲁棒。令牌在边界条件下失效，就该修令牌，而不是在组件上打补丁。

### 1.2 章法：两坨 + 死白，立轴没落实

现状 `centerbox` 把左组甩到最左、右组甩到最右，center 是空 box，于是中间一大段真空：

```
[▦·甲乙丙]░░░░░░░░░░░░░░░░░░░░░░░░░░░░░[tray·📶100%↻·日期时辰]
 左(齐)            死白(画心被掏空)              右坨(挤+吵)
```

压角太重、留白太死，像两堆东西贴在墙两端，立轴的"压角与留白的平衡"没立住。

### 1.3 右端太吵：时间叠了两层 + 字阶失控

右端有竖排 `Tue / Jul 21` + `巳时 · 09:27` + 铃铛。但"巳时"已经回答了"何时"，竖排日期是**重复的一层时间信息**；且 §4.1 时钟格式只定义了「辰时 · 08:08」，dow/md 竖排是实现时多加的。叠加 bar 内文字已有五档字号（1.14 / 1.00 / 0.95 / 0.90 / 0.74），32–44px 的窄条里塞五层字阶，物理上就是"吵"。

### 1.4 左端语言不齐

九宫格是月白实心图标、无框、"飘"着；紧跟的天干印章是带框方块。第一个元素和后三个不是一族人，节奏不齐。

### 1.5 微观节奏散

「·」颜色 0.45 alpha 蟹壳青，在浅色 bar 上几乎消失；使用不一致（左 1 个、右 2 个、位置随意）。印章间距 8px、模块间距 6px，两套步长，且 6px 不在 §6 的 4px 阶梯（4/8/12/16/24）内——越界。tray 是外部进程画的彩色图标，是色彩纪律管不到的飞地，却裸放着。

## 2. 隐喻：把 bar 当一条装裱横幅

bar 的横向形制是**横披**，但钤印、款识、留白、天杆地杆这套章法语言取自立轴；§4.1 原文亦称"装裱卷轴"。下文为简洁，把这套借用立轴章法的横披统称**立轴隐喻**，不纠结横竖。

| 立轴部件 | bar 对应 |
|---|---|
| 绫边（最沉的装裱边） | bar 底色，α 沉墨 |
| 天杆 / 地杆 | 顶 / 底各一条 1px 天青发丝线 |
| 引首章（压卷首的闲章） | app-grid 按钮 |
| 姓名章（落款处的印） | active workspace 朱砂印 |
| 画心留白 | center 墨空（未着墨的绢，**不是窗**） |
| 款识（落款的字） | 右端时辰 |
| 押角印（款识旁的印） | 通知铃铛 |

两个上位取舍，已拍板：

- **结构隐喻 = 重排立轴**（保留全宽贴顶 + 顶底双发丝线，不动摇 §4.1 的"立轴"定义，只重排信息节奏与治墨骨）。
- **画心取向 = α1 墨空留白**（整条高 alpha 沉墨，center 纯空）。理由：宋画的留白本就是"绢 / 墨的空"，**不是窗**——立轴的绫边从来不该透画，画心是空才合宋意。这同时与"治墨骨"同向（中间也沉）。

> 备选登记（**不在本轮**）：α2 绫边夹画心（bar 拆段、中间降 alpha 真透壁纸，最戏剧但工程重、点击穿透与 picom 交互全是坑）；α3 墨空 + 淡题款（center 点一枚极淡闲章，与克制拉扯、易与印章区冗余）。两者写进 §7 非目标，留作日后想更戏剧时的方向。

## 3. 五段决策

### 3.1 墨骨与令牌 —— L1 由 0.75 → 0.90

合成算式 `result = α·ink + (1−α)·wallpaper`，对浅色高光壁纸 `(250,226,190)` 的鲁棒性：

| α (L1) | 压高光后的 bar 色 | 明度 | 冷暖 (R−B) | 比壁纸(62)沉？ | 隔纱感 |
|---|---|---|---|---|---|
| 0.75 现状 | `(82,76,70)` 实测 | 76 | +12 暖 | ✗ 反亮 | 透太多→染脏 |
| 0.88 | `(53,50,49)` | 51 | +4 微暖 | ✓ | 强 |
| **0.90 选定** | **`(48,46,46)`** | **46** | **+2 近中性** | **✓ 沉 16 阶** | **留 10% 纱** |
| 0.92 | `(44,42,43)` | 43 | +1 中性 | ✓ 沉 19 阶 | 纱感弱 |

α = 0.90 时明度落 46（比壁纸沉 16 阶，绫边重新成为最沉的层），R−B 只剩 2（暖染基本抹平，墨底回归冷灰本相），仍留 10% 透色——背后有画这件事"隔纱可感"，没变成死黑塑料。**spec 标可调区间 0.90–0.92**，最终值以 §6 像素采样复核为准。

**发丝线不动**。验算：天青 `(158,200,192)` @0.35 叠沉墨底 `(26,26,30)` ≈ `(72,87,87)`，对墨底明度差足够，"天杆地杆"在沉骨上反而更挺——骨越沉线越亮，立轴感越强。这是 α 调高的附带红利，**不加阴影**（§4.1 明禁阴影，绫边靠线立骨，加阴影会"悬空"违和）。

**只动 L1，不顺手碰 L2/L3/L4**。bar 是唯一"全宽贴边 + 无阴影框定 + 常驻"的面，对壁纸最敏感，是精准打击点；弹层（0.65/0.55/0.60）是临时浮层，自带 `0 4px 12px` 阴影 + 发丝线，视觉边界已被框死，透一点恰是"隔纱观物"。L2–L4 的浅壁纸鲁棒性登记为 §7 观察项。

**落地**：`song-liquid-glass.md` §2.2 表格 L1 行、§4.1 "底色 bg_base alpha 0.75"；`bar.scss.tmpl` 的 `rgba($bg-base, 0.75)`。三处同步改——令牌边界失效，按"修令牌"处理。

### 3.2 左端章法 —— 引首章 + 姓名章成组

把 app-grid 收编成"引首章"，与天干印章同尺寸成族，`飘+框框框` → `框框框框`。

- **尺寸齐平，防跳动**：四枚方印同宽同高同 3px 圆角同 padding。active 朱砂印**不**放大——朱砂实底已是最重一级，靠色不靠尺寸突出，放大会在切换时跳。呼应 §4.2"三态同 border 防跳动"，尺寸也纳入这条纪律。
- **引首章形制 = 天青发丝线框**：与印章同尺寸同 3px 圆角，套一圈天青发丝线边框，内放月白九宫格图标；hover 时边框转亮 + 底微染天青。左端四枚方印彻底成族。
- **色彩四级**（重→轻）：active = 朱砂实底 + 月白粗字（唯一红）；occupied = 天青发丝线 + 天青字；引首章 = 天青发丝线 + 月白图标（图标实心，故视觉不弱于 occupied）；idle = 蟹壳青淡线 + 蟹壳青淡字。
- **左端组内零分隔符**：成组后引首章直接挨印章（同族，靠 8px 步长），现状 `app-grid · 印章` 的「·」**删掉**。
- **步长 8px**：引首章与首枚印章、印章之间，全用 8px（在 §6 阶梯内，保留现状印章间距）。

**落地**：`bar.yuck.tmpl`（app-grid 包进与印章同构的 eventbox/box，删左端「·」）；`bar.scss.tmpl`（`.app-grid` 加天青发丝线框 + hover，与 `.ws-seal` 同尺寸 token）。

### 3.3 右端章法与落款 —— 款识 + 押角印

右端从一堆零件收成"三组 + 落款"：

```
[▢tray▢]  ·  [📶  🔋100%  󰚰•  󰘵]  ·  [ 巳时 · 09:27   🔔• ]
 飞地容器      状态簇(8px,无·)            落款款识      押角印
```

- **砍竖排 dow/md**：实现时多加的一层，§4.1 时钟格式只有「辰时·08:08」。竖排两行删掉，完整日期下沉到 calendar-popup（点时辰本就弹日历）与锁屏。
- **落款 = 纯时辰款识**，字体改**思源宋体** + letter-spacing 2px（§3"标题/时钟展示 = 思源宋体"，bar 时辰正是时钟展示；宋刻本筋骨让这串字读作款识而非读数）。这是 bar 里思源宋体**唯一**出现处，强化落款隐喻。
- **bell = 押角印**：贴时辰右侧。平时蟹壳青淡印（静），未读时角标鲜亮 = 印泥新钤（短暂破静，提醒后即归静）。未读角标仍落屏幕最右角，符合"角标在角"惯例。
- **状态簇内部不用「·」**：net/bat/upd/cc 同族 = 系统状态，靠 8px 步长挨着。
- **update 角标 + notif 角标 双藤黄方点合规**：藤黄是 warn 语义，多处 warn 不冲突；唯一化的是朱砂，不是藤黄。

**落地**：`bar.yuck.tmpl`（删 `.time-dow`/`.time-md` 两个 label 与其 v-box，时辰 label 单留）；`bar.scss.tmpl`（`.time-main` 改 `font-family: "Source Han Serif CN", "Noto Serif CJK SC", serif` + `letter-spacing: 2px`；删 `.time-dow`/`.time-md` 规则）。

### 3.4 分隔符纪律 / 间距网格 / tray 飞地

- **「·」只许出现在语义大分组之间**。重排后全 bar 的分组分隔符「·」只剩右端两个：`[tray] · [状态簇] · [落款]`。左端印章组内、状态簇内、组与留白之间，一律不用「·」。（落款款识内「辰时 · 08:08」的间隔号是 §4.1 时钟格式的字面组成，**不是**分组分隔符，不在此纪律内——故视觉上落款里还有一个间隔号，但分组分隔符「·」确为两处。）
- **「·」退成隐约界格**：alpha 0.45 → **0.32**。它是分隔不是内容，在沉墨底上 0.45 稍重，0.32 才像宣纸上压的一道淡界栏。
- **组内步长统一 8px，纠正 6px 越界**。状态簇内、印章组内、引首章与首印之间，全归 8px。bar 水平 padding 12px 在阶梯内，保留。
- **tray = 唯一色彩飞地，必须收容**。套一个**浅一阶墨色容器** `rgba($ink, 0.30)` + 3px 圆角 + 4px 内 padding，把跳色图标"装进墨盒"。同步在 §7 登记：tray 图标色**不受令牌约束**，但**必须被墨色容器收容**——这是飞地的纪律，不是放任。

**落地**：`bar.scss.tmpl`（`.bar-sep` alpha 0.45→0.32；`.bar-group`/状态簇 spacing 6→8；`.bar-tray` 加墨色容器背景 + 圆角 + 内 padding）；`bar.yuck.tmpl`（右端「·」位置按三组重排，删多余「·」）；`song-liquid-glass.md` §4.1 / §6 / §7。

### 3.5 字阶压平 + 反馈层

**字阶收口为两层**：

- **落款档** —— 仅时辰款识。思源宋体、`fontSize × 1.14`（基数 = `$eww.fontSize`，可调区间 1.10–1.15）、bold、letter-spacing 2px。bar 里唯一的"展示字"。
- **静默档** —— 其余一切文字归 `fontSize × 1.00`、常规字重、无字距：印章天干字、电池百分比、分隔符「·」。废除 0.95 / 0.90 / 0.74 三档。

三个"看着像第三层、其实不是"的界定，防误判：① active 印章的 bold 是**同字号下的字重状态**（且只一枚），是状态语义不是层级；② update / notif 角标是**色块**不是字；③ 图标尺寸 `iconSize` 是 Symbols Nerd Font 的图形尺寸，**单列**，不计入文字字阶。所以"两层"成立。

**反馈层 —— 让立轴"活"，但贴面不悬空**。保留现状 `module-btn` 120ms hover 染底、印章 180ms transition、`urgent-pulse` 脉冲，并补齐：

- 引首章 hover：天青框转亮 + 底微染天青（与印章 hover 同语言）。
- 落款款识 hover：月白提一档；押角印随铃态呼应。
- 状态图标 hover：蟹壳青 → 月白。

全部 120–180ms、**只动色与 1px 边框，不加位移不加阴影**——宋式的"活"是墨色深浅的呼吸，不是弹跳。`urgent-pulse` 是唯一允许的循环动效，且只落在 urgent 印章一枚上。

**落地**：`bar.scss.tmpl`（`.battery-pct` 0.95→1.00；`.bar-sep` 0.90→1.00；删 `.time-dow`/`.time-md` 0.74 档；补 `.app-grid`/`.time-main`/`.module-icon` 的 hover 过渡）。

## 6. 验证闭环（改完必跑）

呼应 §10 像素采样纪律。改完 `chezmoi apply && eww reload` 后：

```bash
import -window root /tmp/bar.png
# 取 bar 底色与正下方壁纸，判墨骨是否立住
magick /tmp/bar.png -format "bar=%[pixel:p{900,12}] wall=%[pixel:p{900,400}]\n" info:
```
